# Plan Lite

Use this skill when a task needs structure, but a full formal plan would be excessive.

## Output Contract
- Goal: one sentence.
- Scope: in/out bullets.
- Steps: 3-7 numbered actions.
- Validation: exact checks to run.
- Risks: top 1-3 risks only.

## Planning Rules
- Prefer smallest deliverable first.
- Include exact file paths when known.
- Keep each step independently verifiable.
- Avoid speculative architecture work.

## Escalate to Superpowers
Escalate to `superpowers:writing-plans` when:
- Work spans multiple subsystems.
- Team handoff requires exhaustive detail.
- Compliance or migration risk is high.

## Terse Mode
- No long preamble.
- Keep total plan under ~200 words unless asked.
- Use compact lists and concrete commands.
